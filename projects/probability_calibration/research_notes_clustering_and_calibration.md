## 1. 现在已经做出来的东西

### 1.1 Checkpoint Brier Score

我们已经对固定 checkpoint 做了 Brier Score：

- 30s
- 60s
- 90s
- 120s
- 180s
- 240s
- 270s

并且对最近 `n` 轮（例如 10 / 100 / 1000）求了平均值。

这个结果是合理的，因为它表达的是：

\[
\frac{1}{N}\sum_{i=1}^{N}(p_{i,t} - y_i)^2
\]

其中：

- `p_{i,t}` = 第 `i` 轮在 checkpoint `t` 时的概率
- `y_i` = 第 `i` 轮最终结果

### 1.2 目前得到的主要结论

对 1000 轮的结果看，Brier Score 大体呈现：

- 早盘较高
- 越接近结算越低

这说明：

> 市场概率会随着时间推进逐步收敛，越接近结算越接近最终结果。

### 1.3 UTC hour slice

也已经做了按 UTC 小时的 mean Brier Score。

结论：

- 不同时段确实有差异
- 但差异属于 **modest but persistent**
- 大约在 `0.16 ~ 0.19` 这个区间波动

所以更稳妥的说法是：

> 市场概率质量存在日内时段差异，但不是极端断层式差异。

不能直接下结论说：

- “这个小时赌徒更多”
- “这个小时一定更好交易”

更稳妥的说法是：

> 某些时段市场更低效 / 更噪声 / 更不准。

## 2. Reliability diagram / calibration bins 是什么

这是 calibration 研究里最核心的图。

它回答的问题是：

> 当市场说“UP 有 70% 概率赢”的时候，现实里它是不是大约真的有 70% 赢？

### 2.1 做法

把概率分箱，例如：

- 0.0–0.1
- 0.1–0.2
- ...
- 0.9–1.0

然后对每个 bin 计算：

1. `mean_predicted_prob`
2. `actual_up_rate`
3. `calibration_gap = mean_predicted_prob - actual_up_rate`
4. `mean_brier_score`
5. `count`

### 2.2 Reliability diagram 的读法

- 横轴 = 平均预测概率
- 纵轴 = 实际发生频率
- 45 度线 = 完美 calibration

如果点：

- 靠近对角线 → 校准好
- 在对角线下方 → 市场高估
- 在对角线上方 → 市场低估

### 2.3 和 Brier Score 的区别

**Brier Score** 问的是：

> 平均错多少？

**Reliability diagram** 问的是：

> 你说 70% 的时候，到底是不是真的 70%？

## 3. 平均概率路径图是什么意思

平均路径图的思路是：

1. 每轮市场都按固定时间对齐
2. 比如每 5 秒采样一次
3. 得到：

\[
(p_0, p_5, p_{10}, ..., p_{300})
\]

4. 再跨很多轮求平均

最后得到一条：

> 市场平均在 5 分钟内如何演化的概率路径

这张图很适合回答：

- 市场平均怎么从开盘走到结算？
- 收敛是平滑发生的，还是后程突然加速？
- 前期是否长期围绕 0.5 犹豫？

## 4. 关于聚类：今天想清楚的核心

### 4.1 原始想法

想做的是：

> 用无监督学习识别不同类型的概率路径图像

这是一个非常好的方向。

目标不是只看：

- 准不准

而是看：

- 市场通常会走出哪些不同的价格发现轨迹

### 4.2 为什么第一版聚类“不太行”

第一次直接把原始路径喂给 k-means：

\[
(p_0, p_5, ..., p_{300})
\]

聚出来的结果基本变成了：

1. 一路向上
2. 一路向下
3. 中间震荡

这说明算法抓住的是：

> 最终方向 / 最终水平

而不是：

- 早收敛
- 晚收敛
- 震荡
- 反转

### 4.3 关键原因

因为 k-means 很容易先抓最强信号。

在这个问题里，最强信号就是：

- 最后接近 1
- 最后接近 0

所以它“偷懒”了，没有真正学路径形状。

---

## 5. 正确改进方向

### 5.1 第一条：先按最终方向分开

必须先拆开：

- 最终 `UP`
- 最终 `DOWN`

因为如果不拆，它们天然一一对应，会强烈主导聚类。

更合理的做法是：

- 先只聚最终 UP 的轮次
- 再单独聚最终 DOWN 的轮次

这样算法才会在组内进一步区分：

- 早收敛上涨型
- 后程拉升型
- 中途震荡后上涨型

### 5.2 第二条：更关注“形状”，少关注“终局水平”

可以考虑把输入改成相对路径，比如：

\[
(p_0-p_0,\ p_5-p_0,\ ..., p_{300}-p_0)
\]

这样更关注：

- 怎么走

而不是：

- 最后在哪

## 6. 什么叫“把路径变成向量”

这是聚类的基础。

### 6.1 最朴素理解

如果一轮市场有很多 tick，那么整条路径可以想成：

\[
(0.50, 0.52, 0.49, 0.53, ..., 0.98)
\]

这就是一个向量。

### 6.2 但不建议直接按 tick 编号对齐

因为不同轮次：

- tick 数不一样
- tick 时间间隔不一样
- 第 100 个 tick 不一定对应同样时间位置

### 6.3 更好的方式：按固定时间采样

比如每 5 秒取一次：

\[
(p_0, p_5, p_{10}, ..., p_{300})
\]

这时每轮都变成长度一致的 61 维向量。

这才是更适合聚类的表示。

## 7. 什么是“特征工程”，为什么它重要

### 7.1 核心理解

特征工程不是直接喂原始路径，

而是先把一条路径总结成若干“有意义的摘要指标”，再拿这些指标组成向量。

也就是：

原来喂的是：

\[
(p_0, p_5, ..., p_{300})
\]

现在喂的是：

\[
(p_{30}, p_{120}, p_{270}, \max p, \min p, \text{波动率}, \text{跨越次数}, \text{首次突破时间}, ...)
\]

### 7.2 为什么这样更可能聚出你想要的东西

因为你真正关心的是：

- 收敛快不快
- 波动大不大
- 有没有反转
- 是否长期围绕 0.5 震荡

这些东西虽然藏在原始路径里，但不够突出。

如果你把它们显式提成特征，聚类就更容易抓到“路径类型”。

### 7.3 对这个项目最有意思的特征

#### 位置特征

- `p_30`
- `p_60`
- `p_120`
- `p_180`
- `p_240`
- `p_270`

#### 变化特征

- `p_270 - p_30`
- `p_180 - p_60`
- 最后 60 秒变化量
- 前 60 秒变化量

#### 波动特征

- 全程标准差
- 前 120 秒标准差
- 最大振幅 `max - min`
- 相邻采样点变化绝对值总和

#### 结构特征

- 是否跨过 0.5
- 跨过 0.5 几次
- 首次突破 0.6 的时间
- 首次突破 0.7 的时间
- 首次进入极端区（如 >0.9 或 <0.1）的时间

#### 收敛特征

- 最早稳定在 >0.8 的时间
- 最早稳定在 <0.2 的时间
- 最后 30 秒是否已稳定
- 距离终值的平均偏差

## 8. 关于“正交化”和标准化

### 8.1 不必一开始追求严格正交化

这里更重要的是：

> 做出有解释力的特征向量

而不是先追求严格数学上的正交成分。

所以现阶段更准确的目标是：

> 喂给聚类算法一组可解释、能区分路径形状的特征向量

### 8.2 权重为什么重要

k-means 用的是距离。

如果你把这些东西混在一起：

- `p_30`（0 到 1）
- `p_270`（0 到 1）
- `time_to_0.7`（0 到 300）
- `cross_count`（0 到若干整数）

那不做处理的话，大尺度特征会天然更主导距离。

### 8.3 最基本处理：标准化

做法：

\[
z = \frac{x - \mu}{\sigma}
\]

也就是：

- 减去均值
- 除以标准差

R 里就是：

```r
X_scaled <- scale(X)
```

### 8.4 什么时候必须标准化

如果你喂的是“混合特征向量”，例如：

- 路径值
- 收敛时间
- 波动率
- 跨越次数

那基本就应该标准化。

### 8.5 什么时候可以先不标准化

如果你喂的是纯路径值：

\[
(p_0, p_5, ..., p_{300})
\]

而且全是同一类变量，可以先试试看不标准化。

但即便如此，也要记住：

> 即使都在 0–1 范围内，不同维度在样本中的波动程度不同，也会导致某些特征更主导聚类。

### 8.6 权重怎么调

在标准化以后，可以再乘权重：

```r
X_scaled[, "time_to_0.7"] <- X_scaled[, "time_to_0.7"] * 2
X_scaled[, "cross_count"] <- X_scaled[, "cross_count"] * 0.5
```

这表示：

- 更强调 `time_to_0.7`
- 更弱化 `cross_count`

所以权重的含义是：

> 告诉算法你更重视哪种路径差异

## 9. 关于 k-means：最简单的理解

k-means 的核心就是：

> 把相似的东西分在一起

在这个项目里：

- 每一轮市场 = 一个向量
- 很多轮市场 = 一个样本矩阵

然后 k-means 会：

1. 先猜几个中心
2. 每轮分给最近的中心
3. 每一组重新算平均中心
4. 重复直到稳定

所以最后得到的是：

- cluster 1 的典型路径
- cluster 2 的典型路径
- cluster 3 的典型路径
