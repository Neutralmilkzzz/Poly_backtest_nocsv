# Telegram Bot 部署与启动教程

你好！这是一份帮助你从零配置并启动回测分析 Telegram Bot 的保姆级教程。

## 步骤 1：获取你的专属 Chat ID
Bot 必须知道是你在发消息才能响应。你需要抓取你的专属 `TELEGRAM_CHAT_ID`。
为方便获取，我已经为你准备好了一个小程序：

1. 打开电脑的终端 (Terminal / CMD)。
2. 进入到项目主干目录：
   ```bash
   cd Poly_backtest_Final
   ```
3. 运行获取工具：
   ```bash
   python backtest_bot/get_chat_id.py
   ```
4. 此时程序会要求你输入 Token。请粘贴你刚才给我的 `870...` 的那串字符。
5. 此时控制台显示“正在轮询等待新消息...”。
6. **拿起手机，在 Telegram 里找到你的机器人，发一句任意文字过去。**
7. 你的终端屏幕上将立刻弹出一长串数字（这就是你的 `Chat ID`）！把它复制下来。

## 步骤 2：写入配置文件 (.env)
为了安全，我们的 Token 和 Chat ID 不直接写在代码里，而是放在环境变量文件中。

1. 在 `backtest_bot/` 文件夹下，新建一个文件，命名为 `.env`。
   （如果你看到有一个 `.env.example`，可以直接把它重命名或者复制一份为 `.env`）。
2. 使用文本编辑器打开 `.env` 文件，粘贴你对应的信息：
   ```ini
   TELEGRAM_BOT_TOKEN="8703629807:AAH9......" 
   TELEGRAM_CHAT_ID="填写你刚刚复制出来的那串数字"
   
   # 如果你需要代理才能上 Telegram（国内环境需要），取消下一行的注释并填入：
   # PROXY="http://127.0.0.1:7890" 
   ```
3. 保存该文件。

## 步骤 3：构建 FST 数据缓存（推荐）
机器人回测时需要读取大量 CSV 数据。提前把 CSV 转成 FST 缓存格式，回测速度可以提升约 12 倍。

1. 在项目根目录运行：
   ```bash
   Rscript scripts/build_fst_cache.R
   ```
2. 等待脚本完成。它会扫描 data/raw/ 里所有 CSV，合格的转成 FST 存入 data/cache/fst/。
3. 之后每次新增 CSV 数据，只需重新运行这条命令，它会增量处理只转新文件。
4. 如果你跳过这步，机器人仍然可以正常运行，只是会慢一些（自动回退到读 CSV）。

## 步骤 4：正式启动机器人！
一切就绪，现在我们让机器人永驻后台。

1. 在项目主干目录 (`Poly_backtest_Final`) 下，可以直接运行我写好的脚本：
   ```bash
   bash start_bot.sh
   ```
2. 屏幕会提示“启动成功，进程 PID ...”。这意味着机器人已经在后台默默工作了。
3. 如果你想看看 Bot 是不是正常或者有没有报错，可以通过查看日志：
   ```bash
   tail -f logs/telegram_bot.log
   ```

## 步骤 5：享受自动化分析吧！
现在打开 Telegram 对着你的 Bot 发送 `/start`。
它秒回你主菜单，你就可以点这键盘发送指令，比如输入 `11` 来试试最新上线的单因子扫参模块啦！🎉
