## 一、起源：为什么要做这个系统

### 1.1 问题本质

Polymarket 的 5 分钟 BTC 二元期权市场不是自然波动市场，而是一个 **Bot 养蛊场**。市场参与者分三类：

| 角色 | 行为 | 占比 |
|------|------|------|
| 散户 Bot | 跑网格、跟趋势、尾盘追涨 | 多数 |
| 庄家 (Whale) | 利用资金优势操纵盘口，制造假突破，收割散户 Bot | 少数但致命 |
| 我们 | 识别庄家何时在场，跟着庄家一起收割散户 | 目标 |

### 1.2 核心洞察

> **常规策略在正常市场赚钱，在庄家市场亏钱。**
> **反打策略在庄家市场赚钱，在正常市场亏钱。**
> **关键不是策略本身，而是知道什么时候用哪个。**

这就是"探针-跟庄"系统的全部哲学。

## 四、核心发现：中盘反弹信号

### 4.1 什么是"中盘反弹"

在 t=150-240s（一盘的中间时段），如果某一侧的 ask 价格跌到很低（比如 ≤$0.12），说明市场出现了剧烈的单边压制。如果这个被压制的价格随后**反弹到 ≥$0.50**，说明发生了"大逆转"——这正是庄家操盘的典型特征：

```
庄家操盘流程:
1. 中盘(150-240s): 大资金砸低某侧价格 → ask跌至0.10-0.15
2. 散户Bot恐慌止损 / 网格Bot被击穿
3. 庄家在底部吸筹
4. 尾盘(240-285s): 庄家拉升，价格反弹至0.50+
5. 散户Bot高位追涨被套
```

### 4.2 为什么选中盘(150-240s)而不是尾盘(240-285s)

| 窗口 | 最优PnL | Sharpe | 原因 |
|------|---------|--------|------|
| **Mid (150-240s)** | **+$2,999** | **3.29** | 庄家操纵从中盘**开始**，是领先信号 |
| Tail (240-285s) | +$2,562 | 2.53 | 尾盘已经是操纵的**结果**，是滞后信号 |
| Early (0-94s) | +$1,336 | 1.46 | 早期便宜ask是正常市场行为，不是操纵 |

**关键洞察**：中盘观测是**领先指标**——看到中盘出现大逆转，说明庄家正在"上班"，下一盘很可能继续操纵。

### 4.3 为什么用bounce≥0.50而不是结算胜负

原始DipBuy探针判断"这一侧最终结算是否赢了"。但这有两个问题：

1. 很多庄家操纵的盘，虽然出现了大逆转（从0.10拉到0.60），但最终结算并不一定赢（因为对手侧可能更强）
2. 结算是t=300s的事件，我们在t=150-240s就需要做判断

**bounce≥0.50 的含义**：不管最终结算谁赢，只要某侧从≤0.15被打到≥0.50，这个"从地板拉到半场"的行为本身就说明有大资金在操控。

## 五、4-Bucket系统：完整策略逻辑

### 5.1 系统状态机

```
                    ┌─────────────────┐
                    │   每盘开始        │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  计算滚动反弹WR   │
                    │  (过去W盘中,       │
                    │   bounce≥0.50     │
                    │   占比多少?)       │
                    └────────┬────────┘
                             │
                 ┌───────────┴───────────┐
                 │                       │
          WR >= T                  WR < T
          (反弹频繁)               (正常市场)
                 │                       │
         ┌───────▼───────┐      ┌───────▼───────┐
         │  WHALE 模式    │      │  NORMAL 模式   │
         │                │      │                │
         │  关闭: 网格    │      │  运行: 网格    │
         │  关闭: 动量    │      │  运行: 动量    │
         │  启动: 反打    │      │  关闭: 反打    │
         │  (fade动量方向) │      │                │
         └───────────────┘      └───────────────┘
```

### 5.2 四个桶的定义

| 桶 | 市场状态 | 使用策略 | 含义 |
|----|---------|---------|------|
| **B1** | Whale | Fade (反打) | ✅ 庄家期反向操作 → **赚钱** |
| B2 | Whale | Normal (动量+网格) | ❌ 庄家期正常操作 → **亏钱** |
| **B3** | Normal | Normal (动量+网格) | ✅ 正常期常规操作 → **赚钱** |
| B4 | Normal | Fade (反打) | ❌ 正常期反向操作 → **亏钱** |

**最优策略 = B1 + B3** — 庄家期反打，正常期常规。

### 5.3 反打(Fade)策略具体操作

```
正常动量: t=250s, up_mid > 0.55 → 买 UP
反打:     t=250s, up_mid > 0.55 → 买 DOWN (反方向!)

直觉: 庄家制造假趋势引诱散户追涨, 实际会反转
所以: 散户看到"强势UP"就追UP, 我们看到"强势UP"就买DOWN
```

## 八、系统运行流程（实盘逻辑）

### 8.1 每盘执行步骤

```
┌─ 盘开始 (t=0s) ──────────────────────────────────────────────┐
│                                                                │
│  Step 1: 读取历史 — 过去7次"中盘便宜事件"的反弹率              │
│          rolling_wr = 过去7次事件中bounce>=0.50的比例            │
│                                                                │
│  Step 2: 判断市场状态                                          │
│          if rolling_wr >= 0.25 → WHALE模式                     │
│          else → NORMAL模式                                     │
│                                                                │
│  Step 3: NORMAL模式                                            │
│    ├─ t=0-94s:   扫描盘口, 任一侧ask<=0.18 → 买入50 shares    │
│    ├─ t=0-190s:  同侧bid>=0.26 → 卖出 (网格止盈)              │
│    └─ t=250s:    up_mid>0.55→买UP / <0.45→买DOWN (动量)       │
│                                                                │
│  Step 3': WHALE模式                                            │
│    ├─ 关闭网格 (不买不卖)                                      │
│    ├─ 关闭正常动量                                              │
│    └─ t=250s:    up_mid>0.55→买DOWN / <0.45→买UP (反打!)      │
│                                                                │
│  Step 4: 观测中盘信号 (纯观测, 不交易)                         │
│    ├─ t=150-240s: 记录每侧ask最低价                            │
│    └─ 如果min_ask<=0.12: 标记为"事件"                          │
│       此后跟踪该侧bid最高价, 记录是否达到>=0.50                │
│                                                                │
│  Step 5: 结算 (t=300s)                                         │
│    ├─ 计算本盘PnL                                              │
│    └─ 更新探针事件历史 (供下盘Step1使用)                        │
│                                                                │
└──────────────────────────────────────────────────────────────────┘
```

### 8.2 关键细节

**探针是虚拟的、零成本的：**
- 不需要真正买入便宜侧
- 只需要在 t=150-240s **观察盘口数据**
- 记录"是否有侧ask<=0.12"+"之后bid是否>=0.50"
- 这个观察结果影响**下一盘**的策略选择

**反打的具体操作：**
- 正常动量：看到市场偏向UP → 买UP → 赌趋势延续
- 反打：看到市场偏向UP → 买DOWN → 赌庄家制造假趋势，实际会反转
- 两者使用完全相同的信号（t=250s的up_midpoint），只是方向相反

**状态切换的滞后性：**
- 当前盘的中盘观测，影响的是**下一盘**的策略选择
- 这意味着信号有1盘的延迟，但这正好让我们有时间准备
- 滚动窗口W=7意味着至少需要7个"中盘便宜事件"才能开始判断

## 九.5、鲸鱼期策略深度优化（Tick级模拟验证）

### 重大修正：4-Bucket翻转法的缺陷

之前4-bucket分析中 `fade_pnl = -momentum_pnl` 是一个**数学近似**，假设了零点差（即 up_ask + down_ask = 1.0）。

但 Polymarket 实际上 **up_ask + down_ask > 1.0**（盘口点差），这意味着：
- 动量买UP at ask=0.90, fade买DOWN应该at down_ask≈0.15（不是0.10）
- 点差吃掉了翻转利润

**Tick级模拟结果 vs 4-Bucket翻转：**

| 方法 | Whale期PnL | Optimal(全系统) | 差异 |
|------|-----------|----------------|------|
| 4-Bucket翻转(之前) | +$916 | +$3,491 | ⚠️ 高估! |
| **Tick级真实模拟** | **+$186** | **+$2,761** | ✅ 真实值 |

### 37种鲸鱼期策略全面测试

在1,585个whale期回合中测试了5大类策略：

| 类别 | 最佳策略 | Whale PnL | Optimal | Sharpe |
|------|---------|-----------|---------|--------|
| **A1: Fade持有到结算** | **反打动量+结算** | **+$186 ✅** | **+$2,761** | **3.03** |
| A2: Fade+止盈 | TP@0.70 | -$789 | +$1,786 | 2.12 |
| B: DipBuy+止盈 | Buy≤0.05,Sell≥0.50 | -$369 | +$2,206 | 2.90 |
| C: DipBuy持有 | Buy≤0.05,Hold | -$169 | +$2,406 | 3.07 |
| D: DipBuy宽窗口 | Buy≤0.10,Sell≥0.50 | -$1,037 | +$1,538 | 1.92 |

**核心发现**：
1. **Fade-Hold是唯一whale期正收益策略！** 所有DipBuy变体在whale期都亏钱
2. DipBuy亏钱的原因：庄家制造便宜ask≤0.05来诱杀，但不保证那一侧结算赢
3. 止盈反而更差：加TP后fade大幅降低，因为TP限制了赢面但不限制输面

|---------|-------|--------|
| 1. 永远开着(不切换) | +$1,833 | 基线 |
| 2. 检测到鲸鱼→**全停** | +$2,575 | +$742 |
| 3. 检测→停常规+**开fade** | +$2,761 | +$928 |

**结论**：系统80%的价值在于**识别并回避**，20%在于**反打赚钱**。即使完全不做fade、只是whale期停止交易，也能多赚$742。

## 十一、风险与局限

### 11.1 已知风险

| 风险 | 严重度 | 对策 |
|------|--------|------|
| 过拟合 | 中 | 三段验证通过，但19天数据仍偏短 |
| 市场结构变化 | 高 | 庄家可能改变策略，需持续监控 |
| 延迟/滑点 | 中 | 保留20-30%安全边际 |
| 手续费侵蚀 | 低 | 2% taker fee已在预估中考虑 |
| 探针窗口延迟 | 低 | 1盘延迟是设计特性，不是bug |

### 11.2 什么情况下系统会失效

1. **庄家不再玩了** — 中盘不再出现cheap ask → 探针无事件 → 永远NORMAL → 回归纯动量
2. **庄家换手法** — 不再中盘砸盘拉升，改用其他操纵方式 → 探针捕捉不到
3. **动量WR大幅下降** — 如果正常市场动量WR从79%降到60%以下，整个系统根基动摇
4. **所有Bot都学会了反打** — 反打变成共识后，庄家会反反打

### 11.3 监控指标

实盘中应持续监控：
- 动量WR (滚动100盘): 低于70% → 预警
- 探针事件频率: 低于50% → 市场结构可能变了
- WHALE模式PnL: 连续20盘为负 → 考虑暂停

## 十二、研究过程记录

### 12.1 探索路径（按时间顺序）

#### 阶段一：基础策略回测
- 修复btc_diff数据bug → 改用up_midpoint判断结算
- 建立三策略基线: 网格(平)、动量(+$1,833)、DipBuy(-$1,263)
- **关键发现**: DipBuy虽然本身亏钱，但其胜率波动包含庄家信息

#### 阶段二：原始探针系统
- 用DipBuy结算胜率(WR)作为庄家指标
- 原版信号: tail窗口(240-285s), 结算win, W=7, T≥0.30
- 结果: +$3,017, SR=3.28, 10.7% whale
- **问题**: 依赖结算结果 = 需要整盘结束才知道信号

#### 阶段三：Relaxed Probe（本轮核心突破）
- **用户洞察**: "不一定要翻到0.99赢才行，翻到0.40以上就可以"
- V1实现有bug: 看的是整盘min_ask而非特定窗口 → 99.9%触发
- V2修正: 4个时间窗口独立扫描
- **关键发现**: bounce≥0.50 优于 settle win
- **关键发现**: mid窗口(150-240s) 优于 tail窗口(240-285s)

#### 阶段四：4-Bucket验证
- 正式验证4桶分离效果
- B1(whale+fade)=✅赚, B2(whale+normal)=❌亏
- B3(normal+normal)=✅赚, B4(normal+fade)=❌亏
- 信号有效性完全确认

#### 阶段五：敏感性测试（最终优化）
- cheap threshold: 0.12 > 0.15 > 0.10（更严格=更精准）
- bounce threshold: 0.50 最优（0.55接近但不够稳健）
- W=7 最优（5-6太快，9-10太慢）
- T=0.25 最优（离散WR: 2/7=28.6%，3/7=42.9%）
- 仓位线性缩放，Sharpe不变

### 12.2 失败的尝试

| 尝试 | 结果 | 教训 |
|------|------|------|
| 用btc_diff判断结算 | 40%数据错误 | 永远用midpoint |
| 滚动动量亏损率做信号 | 太弱 | 动量79%WR太高，亏损太分散 |
| 连亏N盘做触发 | 从未发生 | 连亏5盘在2647盘里概率为零 |
| 网格探针 | 弱于DipBuy | DipBuy直接捕捉价格极端行为 |
| 盘内实时信号(spread/swing) | 最好+$49 | 盘内信号太噪 |
| 早期窗口(0-94s)观测 | +$1,336 | 早期便宜ask是正常的，不是庄家 |
| V1全盘min_ask | 99.9%触发 | 不分时段=无差别=无信息 |

### 12.3 未实现的想法（后续可探索）

1. **探针实盘均线法**: 用实盘网格策略收益率vs历史均线来检测庄家，偏离过大触发WHALE
2. **双窗口确认**: mid + tail 同时出现大逆转才触发，更保守
3. **时段过滤叠加**: 只在动量WR≥78%的小时段交易（之前测过有效，但增加了复杂度）
4. **"小学生策略"深度版**: 当识别到庄家操纵时，直接买入被砸低的那一侧（而不仅是反打动量）
